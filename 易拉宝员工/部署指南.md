# 易拉宝设计助手 - 完整部署指南

## 📋 项目架构

- **后端**: FastAPI (Python) - 端口 8000
- **前端**: Vue 3 + Vite + Element Plus
- **依赖服务**: Claude API, RunningHub API

---

## 🚀 方案一：Railway 部署（推荐 - 最简单）

### 优点
- 零配置，自动检测项目类型
- 免费额度 $5/月
- 自动 HTTPS
- 支持自定义域名

### 步骤

#### 1. 推送代码到 GitHub

```bash
cd 易拉宝员工
git init
git add .
git commit -m "Initial commit"

# 在 GitHub 创建仓库后
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

#### 2. 部署到 Railway

1. 访问 https://railway.app
2. 用 GitHub 登录
3. 点击 "New Project" → "Deploy from GitHub repo"
4. 选择你的仓库
5. Railway 会自动检测并部署

#### 3. 配置环境变量

在 Railway 项目设置中添加：

```
CLAUDE_API_KEY=你的密钥
RUNNINGHUB_API_KEY=你的密钥
CLAUDE_BASE_URL=你的API地址（可选）
```

#### 4. 获取部署地址

部署完成后，Railway 会提供一个域名，如：
`https://your-app-production.up.railway.app`

## 🐳 方案二：Docker 部署（推荐 - 最灵活）

### 优点
- 可部署到任何支持 Docker 的服务器
- 环境一致性好
- 易于迁移

### 步骤

#### 1. 创建 Dockerfile

在 `易拉宝员工` 目录创建 `Dockerfile`:

```dockerfile
# 多阶段构建
FROM node:18-alpine AS frontend-builder

WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm install
COPY frontend/ ./
RUN npm run build

# Python 后端
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY backend/requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt

# 复制后端代码
COPY backend/ ./backend/
COPY *.py ./
COPY knowledge_base/ ./knowledge_base/
COPY logo_library/ ./logo_library/

# 复制前端构建产物
COPY --from=frontend-builder /app/frontend/dist ./frontend/dist

# 创建必要的目录
RUN mkdir -p results temp_uploads knowledge_base/files

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "-m", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

#### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - CLAUDE_API_KEY=${CLAUDE_API_KEY}
      - RUNNINGHUB_API_KEY=${RUNNINGHUB_API_KEY}
      - CLAUDE_BASE_URL=${CLAUDE_BASE_URL}
    volumes:
      - ./results:/app/results
      - ./temp_uploads:/app/temp_uploads
      - ./knowledge_base:/app/knowledge_base
    restart: unless-stopped
```

#### 3. 部署

```bash
# 本地测试
docker-compose up -d

# 部署到服务器
# 1. 上传代码到服务器
# 2. 在服务器上运行
docker-compose up -d
```

## ☁️ 方案三：云服务器部署（传统方式）

### 适用于：阿里云、腾讯云、AWS EC2 等

### 步骤

#### 1. 准备服务器

- 系统：Ubuntu 20.04+ / CentOS 7+
- 配置：2核4G 起步
- 开放端口：80, 443, 8000

#### 2. 安装依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python 3.11
sudo apt install python3.11 python3.11-venv python3-pip -y

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install nodejs -y

# 安装 Nginx
sudo apt install nginx -y
```

#### 3. 部署后端

```bash
# 上传代码到服务器
cd /var/www/banner-designer

# 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
cd 易拉宝员工
pip install -r backend/requirements.txt

# 配置环境变量
cp .env.example .env
nano .env  # 编辑填入你的 API Keys

# 创建必要目录
mkdir -p results temp_uploads knowledge_base/files

# 使用 systemd 管理服务
sudo nano /etc/systemd/system/banner-backend.service
```

**banner-backend.service 内容：**

```ini
[Unit]
Description=Banner Designer Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/banner-designer/易拉宝员工
Environment="PATH=/var/www/banner-designer/venv/bin"
ExecStart=/var/www/banner-designer/venv/bin/uvicorn backend.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable banner-backend
sudo systemctl start banner-backend
```

#### 4. 部署前端

```bash
cd frontend
npm install
npm run build

# 将构建产物复制到 Nginx 目录
sudo cp -r dist /var/www/html/banner-designer
```

#### 5. 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/banner-designer
```

**Nginx 配置：**

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 改成你的域名或 IP

    # 前端
    location / {
        root /var/www/html/banner-designer;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态文件
    location /results {
        alias /var/www/banner-designer/易拉宝员工/results;
    }

    location /knowledge_base {
        alias /var/www/banner-designer/易拉宝员工/knowledge_base;
    }

    location /logo_library {
        alias /var/www/banner-designer/易拉宝员工/logo_library;
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/banner-designer /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 6. 配置 HTTPS（可选但推荐）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d your-domain.com
```

## 🔧 前端 API 地址配置

部署后需要修改前端的 API 地址：

### 方法一：环境变量（推荐）

在 `frontend/` 创建 `.env.production`:

```
VITE_API_BASE_URL=https://your-domain.com
```

修改前端代码使用环境变量：

```javascript
// frontend/src/api/chat.js
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000';
```

### 方法二：直接修改

找到前端代码中的 API 地址配置，改为你的服务器地址。

## ⚠️ 重要注意事项

### 1. 环境变量安全
- **不要**将 `.env` 文件提交到 Git
- 在 `.gitignore` 中确保已排除 `.env`

### 2. 文件存储
- Railway/Docker 重启后临时文件会丢失
- 生产环境建议使用对象存储（阿里云 OSS / AWS S3）

### 3. 数据持久化
- 当前使用 JSON 文件存储数据
- 生产环境建议使用 PostgreSQL / MySQL

### 4. 性能优化
- 启用 Nginx gzip 压缩
- 配置静态资源缓存
- 使用 CDN 加速

## 🐛 常见问题

### 1. 端口被占用
```bash
# 查看端口占用
sudo lsof -i :8000
# 杀死进程
sudo kill -9 PID
```

### 2. 权限问题
```bash
# 给目录写权限
sudo chown -R www-data:www-data /var/www/banner-designer
sudo chmod -R 755 /var/www/banner-designer
```

### 3. API 调用失败
- 检查环境变量是否正确
- 查看后端日志：`sudo journalctl -u banner-backend -f`
- 检查防火墙设置

### 4. 跨域问题
- 确保后端 CORS 配置正确
- 检查 Nginx 代理配置
