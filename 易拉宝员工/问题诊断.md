## 🎯 问题 1：生成图片失败（最重要）

### 可能原因：

#### A. Railway 环境变量未配置
Railway 后端需要这些 API Keys：
- `CLAUDE_API_KEY`
- `RUNNINGHUB_API_KEY`
- `CLAUDE_BASE_URL`（如果使用代理）

**检查步骤：**
1. 进入 Railway 项目
2. 点击 **Settings** → **Variables**
3. 确认是否有这 3 个环境变量
4. 如果没有，点击 **New Variable** 添加

#### B. API Key 无效或额度不足
- Claude API Key 是否有效？
- RunningHub API Key 是否有效？
- 账户是否有余额？

#### C. 后端代码问题
查看 Railway 的运行日志：
1. 在 Railway 点击 **Deployments**
2. 点击最新的部署
3. 点击 **View Logs**
4. 查找错误信息（特别是生成图片时的错误）

## 🎯 问题 2 & 3：历史记录和知识库失败

### 原因：Railway 文件系统是临时的

Railway 的文件系统在每次重启后会重置，所以：
- `generation_history.json` 不存在
- `knowledge_base/products.json` 不存在

### 解决方案：

#### 方案 A：修改后端代码，初始化空文件

在 `backend/main.py` 的启动部分添加：

```python
# 在文件开头，app 初始化之后添加
@app.on_event("startup")
async def startup_event():
    """应用启动时初始化必要的文件和目录"""
    # 创建必要的目录
    Path("results").mkdir(exist_ok=True)
    Path("temp_uploads").mkdir(exist_ok=True)
    Path("knowledge_base/files").mkdir(parents=True, exist_ok=True)
    
    # 初始化 generation_history.json
    if not GENERATION_HISTORY_FILE.exists():
        GENERATION_HISTORY_FILE.write_text("[]", encoding="utf-8")
        print("[INFO] 初始化 generation_history.json")
    
    # 初始化知识库
    kb_file = Path("knowledge_base/products.json")
    if not kb_file.exists():
        kb_file.write_text("[]", encoding="utf-8")
        print("[INFO] 初始化 knowledge_base/products.json")
```

#### 方案 B：使用数据库（生产环境推荐）

Railway 提供免费的 PostgreSQL，可以持久化存储数据。
