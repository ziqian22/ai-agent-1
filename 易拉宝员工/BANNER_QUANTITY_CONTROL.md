# 易拉宝生成数量控制

## 更新时间
2026-05-18

## 功能说明

用户可以指定生成易拉宝的数量，系统会根据用户的要求生成相应数量的图片。

## 默认行为

- **默认数量**: 5 张
- **数量范围**: 1-10 张（自动限制）

## 用户如何指定数量

### 方式 1: 明确说明数量
```
用户: "生成 3 张易拉宝"
用户: "给我 8 张"
用户: "我要 10 张不同风格的"
```

### 方式 2: 在对话中提及
```
用户: "帮我设计易拉宝，要 7 张"
用户: "生成易拉宝，数量 6 张"
```

### 方式 3: 不指定（使用默认）
```
用户: "生成易拉宝"
用户: "开始设计"
→ 系统自动生成 5 张
```

## Claude 的处理逻辑

Claude 会从用户对话中提取数量信息：

```json
[GENERATE_BANNER]
{
  "product_name": "朴道健康水",
  "brand": "朴道",
  "slogan": "健康水专家",
  "features": ["...", "...", "..."],
  "scenes": ["...", "..."],
  "style": "科技感",
  "main_colors": "蓝色+白色",
  "num_images": 3  // 用户指定的数量
}
[/GENERATE_BANNER]
```

## 后端处理

```python
# 获取生成数量（默认5张）
num_images = generation_params.get("num_images", 5)

# 确保数量在合理范围内（1-10张）
num_images = max(1, min(10, num_images))

# 调用生成 API
result = await banner_generator.generate_complete_flow(
    ...
    num_images=num_images
)
```

## Logo 处理

无论生成多少张易拉宝，Logo 处理逻辑保持不变：

1. 只分析第一张易拉宝
2. 确定 Logo 方案（Logo ID、位置、尺寸）
3. 将相同的 Logo 应用到所有图片

## 生成时间估算

- **1 张**: 约 1-2 分钟
- **3 张**: 约 3-4 分钟
- **5 张**: 约 6-8 分钟（默认）
- **8 张**: 约 10-12 分钟
- **10 张**: 约 12-15 分钟

## 注意事项

1. **数量限制**: 系统自动限制在 1-10 张之间
   - 如果用户说 "生成 20 张"，系统会限制为 10 张
   - 如果用户说 "生成 0 张"，系统会设为 1 张

2. **生成时间**: 数量越多，等待时间越长
   - 系统会并发生成，但仍需等待所有任务完成

3. **Logo 一致性**: 所有图片使用相同的 Logo
   - 只分析第一张，其他都应用相同配置

4. **API 成本**: 
   - 生成 API: 按数量计费
   - Claude Vision API: 只调用 1 次（分析第一张）
