# 前端快捷按钮功能说明

## 修改日期
2026-05-18

## 功能描述
在 Claude 的回复消息中自动显示快捷按钮，让用户可以一键确认或修改信息，无需手动输入文字。

---

## 修改文件
`frontend/src/components/ChatArea.vue`

---

## 功能实现

### 1. 按钮显示逻辑

当 Claude 的回复中包含以下关键词时，会自动显示快捷按钮：
- "请确认以上信息"
- "确认无误后"
- "请点击"
- "开始生成"
- "信息收集完成"
- "✅ 信息收集完成"

### 2. 按钮样式

按钮显示在 Claude 消息框的**内部底部**，样式如下：
- 与消息内容用分隔线隔开
- 两个按钮：
  - **✅ 开始生成**（主按钮，蓝色）
  - **✏️ 修改信息**（次要按钮，灰色）

### 3. 按钮功能

#### ✅ 开始生成
- 点击后自动发送："确认，开始生成"
- 触发 Claude 输出 `[GENERATE_BANNER]` 指令
- 后端开始生成易拉宝

#### ✏️ 修改信息
- 点击后自动发送："我需要修改信息"
- Claude 会询问需要修改哪些信息
- 用户可以继续对话修改

## 代码实现

### 模板部分（Template）

```vue
<!-- 快捷按钮（在消息内部） -->
<div v-if="msg.role === 'assistant' && shouldShowConfirmButton(msg.content)" 
     class="quick-actions-inline">
  <el-button
    type="primary"
    size="default"
    @click="sendQuickMessage('确认，开始生成')"
    :disabled="loading"
  >
    ✅ 开始生成
  </el-button>
  <el-button
    size="default"
    @click="sendQuickMessage('我需要修改信息')"
    :disabled="loading"
  >
    ✏️ 修改信息
  </el-button>
</div>
```

### 脚本部分（Script）

```javascript
// 发送快捷消息（点击按钮时）
const sendQuickMessage = async (message) => {
  if (loading.value) return

  // 添加用户消息
  messages.value.push({
    role: 'user',
    content: message
  })

  scrollToBottom()
  loading.value = true

  try {
    const response = await sendChatMessage(sessionId.value, message)

    // 添加助手回复
    messages.value.push({
      role: 'assistant',
      content: response.content,
      images: response.images
    })

    scrollToBottom()
  } catch (error) {
    ElMessage.error(error.message || '发送失败')
  } finally {
    loading.value = false
  }
}

// 判断是否应该显示确认按钮
const shouldShowConfirmButton = (content) => {
  if (!content) return false

  // 检查是否包含确认提示的关键词
  const confirmKeywords = [
    '请确认以上信息',
    '确认无误后',
    '请点击',
    '开始生成',
    '信息收集完成',
    '✅ 信息收集完成'
  ]

  return confirmKeywords.some(keyword => content.includes(keyword))
}
```

## 使用流程示例

### 场景1：正常确认流程

1. **用户上传图片** → Claude 分析产品信息
2. **用户提供风格等信息** → Claude 收集信息
3. **Claude 展示信息摘要**：
   ```
   ✅ 信息收集完成
   
   让我确认一下收集到的信息:
   - 产品名称: XXX
   - 品牌: XXX
   ...
   
   请确认以上信息是否正确？
   ```
4. **自动显示按钮**：
   - [✅ 开始生成]
   - [✏️ 修改信息]
5. **用户点击"开始生成"** → 自动发送"确认，开始生成"
6. **Claude 输出生成指令** → 后端开始生成

### 场景2：修改信息流程

1. **Claude 展示信息摘要** + 显示按钮
2. **用户点击"修改信息"** → 自动发送"我需要修改信息"
3. **Claude 询问**："请告诉我需要修改哪些信息？"
4. **用户回复**："品牌改成XXX"
5. **Claude 更新信息并重新展示摘要** + 再次显示按钮
6. **用户点击"开始生成"** → 开始生成

## 优势

1. **用户体验更好**
   - 无需手动输入"确认"、"开始生成"等文字
   - 一键操作，更加直观

   - 按钮在消息框内部，视觉连贯
   - 使用 Emoji 图标，更加友好

4. **响应式设计**
   - 按钮会自动换行（flex-wrap）
   - 适配不同屏幕尺寸

## 注意事项

1. **按钮只显示在最新的助手消息中**
   - 历史消息不会显示按钮
   - 避免用户误点击旧消息的按钮

2. **加载中时按钮禁用**
   - 防止重复点击
   - 提供更好的用户反馈

3. **关键词匹配**
   - 如果 Claude 的回复格式改变，可能需要更新关键词列表
   - 建议 Claude 在需要确认时使用固定的格式
