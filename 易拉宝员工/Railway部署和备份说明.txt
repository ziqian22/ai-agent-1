Railway 部署和自动备份功能说明
===============================================================================

【已实现的功能】

1. 自动备份到 Supabase Storage
   - 每次添加/修改产品时，自动备份 products.json 到 Supabase
   - 备份路径: banner-images/backups/products.json
   - 无需手动操作

2. 启动时自动恢复
   - Railway 重启时，如果本地没有数据，自动从 Supabase 下载
   - 确保数据不丢失

3. 智能判断
   - 如果本地已有数据，不会覆盖（避免误删除）
   - 如果 Supabase 没有备份，继续使用本地数据

===============================================================================

启动流程:
  1. Railway 启动容器

  1. Railway 重启（容器销毁）
  2. 新容器启动
  3. 自动从 Supabase 下载 products.json
  4. 数据恢复完成 ✓

【注意事项】
===============================================================================

1. Supabase Storage 必须配置为 Public
   - 否则无法下载备份文件
   - 在 Supabase Dashboard 中设置

2. products.json 不需要手动备份
   - 每次添加产品都会自动备份
   - 不会遗漏任何数据

3. Railway 环境变量
   - 确保在 Railway 控制台也配置了 Supabase 环境变量
   - 与本地 .env 保持一致

4. 备份频率
   - 每次修改立即备份
   - 不需要定时任务

5. 恢复策略
   - 只在本地无数据时恢复
   - 不会覆盖现有数据

===============================================================================
【故障排除】
===============================================================================

问题1: Railway 重启后数据还是丢失
  原因: Supabase 环境变量未配置
  解决: 在 Railway 控制台添加环境变量

问题2: 备份失败
  原因: Bucket 权限不足
  解决: 设置 banner-images bucket 为 Public

问题3: 恢复失败
  原因: 网络问题或备份文件不存在
  解决: 检查 Supabase 连接和备份文件
