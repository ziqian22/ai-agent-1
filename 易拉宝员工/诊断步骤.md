# Railway 部署问题诊断

## 🔍 当前问题

Railway 构建日志显示：
```
Collecting streamlit==1.31.0 (from -r requirements.txt (line 1))
Collecting pillow==10.2.0 (from -r requirements.txt (line 4))
```

这说明 Railway 还在读取**旧版本**的 requirements.txt 文件。

## ✅ 检查清单

### 1. 确认 GitHub 文件是否真的更新了

访问以下链接，检查文件内容：

**根目录 requirements.txt:**
https://github.com/ziqian22/ai-agent-1/blob/main/易拉宝员工/requirements.txt

应该看到：
- ✅ 第 1 行是 `# 易拉宝设计助手 - 后端依赖`（不是 `streamlit==1.31.0`）
- ✅ pillow 那行是 `pillow>=10.3.0`（不是 `pillow==10.2.0`）

**backend/requirements.txt:**
https://github.com/ziqian22/ai-agent-1/blob/main/易拉宝员工/backend/requirements.txt

应该看到：
- ✅ 第 5 行是 `pillow>=10.3.0`（不是 `pillow==10.2.0`）

### 2. 如果文件没有更新

可能的原因：
- ❌ 编辑后没有点击 "Commit changes"
- ❌ 提交到了错误的分支（不是 main 分支）
- ❌ GitHub 网页编辑器有缓存

**解决方法：重新编辑**

1. 访问 https://github.com/ziqian22/ai-agent-1/blob/main/易拉宝员工/requirements.txt
2. 点击右上角 **铅笔图标** 编辑
3. **删除所有内容**，然后粘贴以下内容：

```txt
fastapi==0.109.0
uvicorn[standard]==0.27.0
httpx==0.26.0
python-dotenv==1.0.0
pillow>=10.3.0
anthropic>=0.18.0
PyPDF2>=3.0.0
python-docx>=1.1.0
python-pptx>=0.6.23
python-multipart==0.0.6
aiofiles==23.2.1
```

4. 点击 **Commit changes**
5. 确认分支是 **main**
6. 点击 **Commit changes** 确认

### 3. 如果文件已经更新但 Railway 还是用旧的

可能的原因：
- Railway 缓存了旧的构建
- Railway 没有正确检测到文件变化

**解决方法：强制重新部署**

在 Railway 控制台：
1. 进入你的项目
2. 点击 "Settings" 标签
3. 滚动到底部，点击 **"Redeploy"**
4. 选择 **"Clear build cache and redeploy"**（清除缓存并重新部署）

### 4. 终极方案：删除项目重新部署

如果以上都不行：

1. 在 Railway 删除当前项目
2. 创建新项目
3. 重新连接 GitHub 仓库
4. 确保选择 **main** 分支
5. 添加环境变量：
   ```
   CLAUDE_API_KEY=你的密钥
   RUNNINGHUB_API_KEY=你的密钥
   ```

## 🎯 推荐操作顺序

1. **先检查** GitHub 文件内容（访问上面的链接）
2. **如果文件是旧的**，重新编辑并提交
3. **如果文件是新的**，在 Railway 点击 "Clear build cache and redeploy"
