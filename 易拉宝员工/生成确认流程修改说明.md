# 生成确认流程修改说明

## 修改日期
2026-05-18

## 问题描述
之前 Claude 在收集完信息后，可能会直接输出 `[GENERATE_BANNER]` 开始生成，没有等待用户明确确认。

## 修改内容

### 修改文件
`backend/main.py` - System Prompt

### 新增的确认流程

#### 第1步：展示信息摘要
当信息收集完成后，Claude **必须**向用户展示完整的信息摘要：

```
✅ 信息收集完成

让我确认一下收集到的信息:
- **产品名称**: XXX
- **品牌**: XXX
- **核心卖点**: XXX
- **产品特点**:
  1. XXX
  2. XXX
  3. XXX
- **适用场景**: XXX
- **设计风格**: XXX
- **主色调**: XXX
- **生成数量**: X张

请确认以上信息是否正确？如需修改请告诉我。
确认无误后,请点击"开始生成"按钮,我将为您生成易拉宝。
```

#### 第2步：等待用户确认
- ❌ **绝对不要**在展示摘要后立即输出 `[GENERATE_BANNER]`
- ✅ **必须等待**用户明确回复：
  - "确认"
  - "开始生成"
  - "没问题"
  - "可以"
  - 等确认词语
- 如果用户提出修改，先修改信息，再重新展示摘要
- 只有在用户明确确认后，才能进入第3步

#### 第3步：输出生成指令
只有在用户明确确认后，才输出：

```
[GENERATE_BANNER]
{
  "product_name": "产品名称",
  "brand": "品牌名称",
  "slogan": "核心卖点",
  "features": ["特点1", "特点2", "特点3"],
  "scenes": ["场景1", "场景2"],
  "style": "科技感",
  "main_colors": "蓝色+白色",
  "num_images": 5
}
[/GENERATE_BANNER]
```

---

## 禁止行为

System Prompt 中明确禁止以下行为：

1. ❌ 禁止在展示信息摘要的同时输出 `[GENERATE_BANNER]`
2. ❌ 禁止在用户未确认的情况下输出 `[GENERATE_BANNER]`
3. ❌ 禁止跳过信息摘要直接生成

## 测试建议

重启后端服务器后，测试以下场景：

1. **正常流程**：上传图片 → 收集信息 → 展示摘要 → 用户确认 → 开始生成
2. **修改信息**：展示摘要 → 用户要求修改 → 重新展示摘要 → 用户确认 → 开始生成
3. **多轮对话**：逐步收集信息 → 展示摘要 → 用户确认 → 开始生成

确保在所有场景下，Claude 都会：
- ✅ 展示完整的信息摘要
- ✅ 等待用户明确确认
- ✅ 只有在确认后才输出 `[GENERATE_BANNER]`

## 总结

通过修改 System Prompt，现在 Claude 会：

1. **主动展示**信息摘要
2. **明确提示**用户确认
3. **等待确认**后才开始生成
4. **禁止跳过**确认步骤

这样可以确保用户在生成前有机会检查和修改信息，避免生成错误的易拉宝。
