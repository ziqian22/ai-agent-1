# 页面使用精灵助手 - 功能说明

## 📌 功能概述

页面使用精灵助手是一个嵌入在系统右下角的智能帮助助手，基于 Claude Haiku 4.5 模型，专门用于指导用户使用系统的各项功能。

## ✨ 特点

1. **智能问答**：了解系统的所有功能，能准确回答用户的问题
2. **诚实透明**：对于系统不支持的功能，会如实告知用户
3. **友好界面**：悬浮球设计，不干扰主功能
4. **快速响应**：使用 Haiku 模型，响应速度快，成本低
5. **对话记忆**：保留最近10条对话历史，上下文连贯

## 🎯 实现方式

### 前端组件
- **HelpAssistant.vue**：悬浮球 + 对话面板
- 位置：右下角固定位置
- 样式：渐变紫色主题，与主界面区分

### 后端 API
- **接口**：`POST /api/help/chat`
- **模型**：`claude-haiku-4-5-20251001`
- **System Prompt**：包含完整的系统功能说明

### API 配置
- 使用环境变量中的 `CLAUDE_API_KEY` 和 `CLAUDE_BASE_URL`
- 云雾 API（yunwu.ai）

## 🚀 使用方式

1. **打开助手**
   - 点击右下角的紫色悬浮球（问号图标）
   - 面板从右侧滑出

2. **提问**
   - 直接在输入框输入问题
   - 点击发送或按 Enter 键
   - 也可以点击快捷问题

3. **查看回答**
   - 助手会实时回复
   - 支持简单的 Markdown 格式
   - 显示消息时间

4. **关闭助手**
   - 点击面板右上角的关闭按钮
   - 或再次点击悬浮球

## 🔧 技术细节

### 前端技术栈
- Vue 3 Composition API
- Element Plus UI 组件
- Axios 网络请求

### 后端技术栈
- FastAPI 框架
- Claude API (Haiku 4.5)
- Pydantic 数据验证

### 性能优化
- 只保留最近10条对话历史（避免 token 浪费）
- 使用 Haiku 模型（响应快，成本低）
- Max tokens 限制为 1024（回答简洁）

## 📊 成本估算

使用 Claude Haiku 4.5 模型：
- 输入：$0.25 / 1M tokens
- 输出：$1.25 / 1M tokens

平均每次对话成本：
- System Prompt: ~2000 tokens
- 对话历史: ~500 tokens
- 用户问题: ~50 tokens
- 助手回答: ~200 tokens

**每次对话成本约：$0.0009（不到1美分）**

## 🎨 界面设计

### 悬浮球
- 大小：60px × 60px
- 颜色：Element Plus 主题蓝色
- 动画：hover 时放大 1.1 倍
- 阴影：柔和投影

### 对话面板
- 尺寸：400px × 600px
- 位置：右下角，悬浮球上方
- 动画：滑入/滑出效果
- 主题：渐变紫色（与主界面区分）

### 消息气泡
- 助手消息：左侧，白色背景
- 用户消息：右侧，蓝色背景
- 打字动画：三个跳动的小圆点

- 不记录敏感信息（API Key、用户数据等）
- 只保留最近10条对话（不永久存储）
- 所有请求通过后端代理（前端不暴露 API Key）
- 对话内容不会用于训练或分析

## 📞 问题排查

**悬浮球不显示？**
- 检查 App.vue 是否导入 HelpAssistant 组件
- 检查组件是否正确注册

**发送消息无响应？**
- 检查后端 API 是否正常运行
- 检查环境变量 CLAUDE_API_KEY 是否配置
- 查看浏览器控制台错误信息

**回答不准确？**
- 检查 System Prompt 是否正确
- 考虑增加更详细的功能说明
- 调整 max_tokens 参数
