## 🚀 下一步：开始部署（只需 3 步）

### 第 1 步：在 GitHub 创建仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `banner-ai-designer`（或你喜欢的名字）
   - **Description**: `易拉宝AI设计助手 - 基于Claude和RunningHub的智能设计工具`
   - **Visibility**: 选择 **Private**（推荐）或 Public
3. ⚠️ **重要**：不要勾选任何选项（README、.gitignore、License）
4. 点击 **"Create repository"**

### 第 1.5 步：推送代码到 GitHub

GitHub 创建完成后会显示推送命令，在终端执行：

```bash
# 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/banner-ai-designer.git

# 推送代码
git push -u origin main
```

**提示**：如果提示需要认证，使用 GitHub Personal Access Token

### 第 2 步：在 Railway 部署

1. 访问 https://railway.app
2. 用 **GitHub 账号登录**
3. 点击 **"New Project"**
4. 选择 **"Deploy from GitHub repo"**
5. 在列表中找到并选择 `banner-ai-designer` 仓库
6. Railway 会自动检测配置并开始部署（约 3-5 分钟）

**配置环境变量**：

部署开始后，立即配置环境变量：

1. 在 Railway 项目页面，点击你的服务
2. 切换到 **"Variables"** 标签
3. 点击 **"New Variable"** 添加以下变量：

```
CLAUDE_API_KEY=你的Claude密钥
RUNNINGHUB_API_KEY=你的RunningHub密钥
CLAUDE_BASE_URL=你的Claude地址（可选，使用官方API可不填）
```

4. 点击 **"Add"** 保存
5. Railway 会自动重新部署

### 第 3 步：获取域名并更新前端配置

**获取 Railway 域名**：

1. 部署完成后，在项目页面点击 **"Settings"**
2. 找到 **"Domains"** 部分
3. 点击 **"Generate Domain"**
4. 复制生成的域名，例如：
   ```
   https://banner-ai-designer-production.up.railway.app
   ```

**更新前端 API 地址**：

1. 在本地编辑 `frontend/.env.production` 文件：
   ```env
   VITE_API_URL=https://banner-ai-designer-production.up.railway.app
   ```
   （替换为你的实际域名）

2. 提交并推送：
   ```bash
   git add frontend/.env.production
   git commit -m "更新生产环境 API 地址"
   git push
   ```

3. Railway 会自动重新部署（约 2-3 分钟）

✅ **完成！** 访问你的 Railway 域名即可使用。

## 🔍 故障排查

### 问题 1：推送到 GitHub 失败

**错误提示**：`Authentication failed` 或 `Permission denied`

**解决方案**：
1. 使用 GitHub Personal Access Token
2. 访问 GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
3. 点击 "Generate new token (classic)"
4. 勾选 `repo` 权限
5. 复制生成的 token
6. 推送时使用 token 作为密码

### 问题 2：Railway 部署失败

**查看日志**：
- Railway 项目页面 → **Deployments** 标签
- 点击失败的部署 → 查看 **Logs**

**常见原因**：
- ❌ 依赖安装失败 → 检查 `backend/requirements.txt` 格式
- ❌ 环境变量未配置 → 检查 **Variables** 是否添加
- ❌ 构建超时 → 等待重试或联系 Railway 支持

### 问题 3：部署成功但无法访问

**检查清单**：
1. ✅ 确认 Railway 已生成域名（Settings → Domains）
2. ✅ 确认环境变量已配置（Variables 标签）
3. ✅ 查看部署日志是否有错误（Deployments → Logs）
4. ✅ 等待 2-3 分钟让服务完全启动

### 问题 4：前端可以访问，但 API 调用失败

**原因**：前端 API 地址未更新

**解决方案**：
1. 编辑 `frontend/.env.production`
2. 将 `VITE_API_URL` 改为你的 Railway 域名
3. 提交并推送代码
4. 等待 Railway 重新部署

### 问题 5：图片无法显示

**原因**：Railway 重启后临时文件丢失

**临时方案**：
- 每次生成的图片会在 Railway 重启后消失
- 建议下载保存到本地

**长期方案**：
- 集成对象存储（AWS S3、阿里云 OSS）
- 将生成的图片上传到云存储
