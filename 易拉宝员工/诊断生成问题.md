# 生成数量问题诊断

## 问题现象

用户要求生成3张易拉宝,但实际只生成了1张。

## 日志分析

```
[DEBUG] 开始生成易拉宝:
  - 产品图片: ...
  - 生成数量: 3  ✅ 参数传递正确
  - 提示词长度: 1251 字符

[上传] 尝试上传图片 (第1/3次)...  ← 产品图上传
[上传] 成功: openapi/62741108...
[上传] 尝试上传图片 (第1/3次)...  ← Logo上传  
[上传] 成功: openapi/57bb3796...

[DEBUG] 生成结果: success=True, error=None
[INFO] 开始为 1 张易拉宝自动添加 Logo...  ❌ 只有1张!
```

## 关键发现

**缺失的日志**:
- 没有看到"正在提交任务 1/3"
- 没有看到"正在提交任务 2/3"  
- 没有看到"正在提交任务 3/3"
- 没有看到"[WAIT] 等待任务 1/3 完成..."

这说明:
1. 循环可能没有执行
2. 或者progress_callback没有被传递
3. 或者任务提交失败但没有记录错误

## 可能原因

### 1. 后端没有重启 ⭐ 最可能
- 修改了banner_generator.py添加num_images参数
- 但后端没有重启
- 还在使用旧代码(没有num_images参数)
- 旧代码默认只生成1张

**验证方法**:
```bash
# 检查后端进程启动时间
ps aux | grep python | grep main.py

# 检查文件修改时间
ls -lh banner_generator.py
```

### 2. API配额限制
- RunningHub API可能限制并发任务数
- 第一个任务成功,后续任务被拒绝
- 但错误没有被正确记录

**验证方法**:
查看完整的后端日志,看是否有API错误

### 3. progress_callback未传递
- backend/main.py调用时没有传递progress_callback
- 导致进度日志没有输出
- 但任务应该还是会提交

**验证方法**:
```python
# 检查backend/main.py第525行
result = await banner_generator.generate_complete_flow(
    ...
    num_images=num_images
    # 缺少 progress_callback 参数?
)
```

## 解决方案

### 方案1: 重启后端 (推荐)
```bash
# 停止当前后端
pkill -f "python.*main.py"

# 重新启动
cd backend
python main.py
```

### 方案2: 检查API配额
联系RunningHub查看API配额限制

### 方案3: 添加更多日志
在banner_generator.py的循环中添加print语句:
```python
for i, variant_prompt in enumerate(variant_prompts):
    print(f"[DEBUG] 正在提交任务 {i+1}/{num_images}")
    task_id = self.generate_banner(...)
    print(f"[DEBUG] 任务 {i+1} task_id: {task_id}")
```

## 关于"时间特别慢"

当前是**串行提交 + 并行等待**:
```python
# 串行提交(慢)
for i in range(3):
    task_id = generate_banner()  # 每次等待API响应
    task_ids.append(task_id)

# 并行等待(快)
for task_id in task_ids:
    wait_for_completion(task_id)  # 同时等待所有任务
```

**改进方案**: 使用真正的并行提交
```python
import asyncio

# 并行提交
tasks = [
    asyncio.create_task(generate_banner_async(...))
    for i in range(3)
]
task_ids = await asyncio.gather(*tasks)
```

但这需要generate_banner改为async函数。
