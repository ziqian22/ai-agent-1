# 风格列表显示修复总结

## 问题描述

用户反馈Claude在引导用户选择风格时,只显示5种风格,缺少第6种"具体场景"风格。

## 根本原因

在多个地方生成风格选择提示时,使用了硬编码的文本,没有动态读取STYLE_PRESETS配置。虽然quick_actions按钮正确显示了所有6种风格,但Claude生成的对话文本中没有包含第6种风格的信息。

## 修复内容

### 1. agent_orchestrator.py

#### 新增函数: `_get_style_list_text()`
- 动态读取STYLE_PRESETS生成风格列表文本
- 自动检测"具体场景"风格并添加预设场景说明
- 位置: 第1469行之前

```python
def _get_style_list_text(self) -> str:
    """生成风格列表文本"""
    style_list = []
    for i, (name, config) in enumerate(STYLE_PRESETS.items(), 1):
        style_desc = f"{i}. {config['icon']} **{name}**: {config['description']}"
        
        # 如果是"具体场景"风格,添加预设场景说明
        if config.get('requires_sub_selection', False):
            suggested_scenes = config.get('suggested_scenes', [])
            if suggested_scenes:
                scenes_text = '\n   '.join([f"• {scene}" for scene in suggested_scenes])
                style_desc += f"\n   预设场景：\n   {scenes_text}\n   （您也可以自己描述想要的场景）"
        
        style_list.append(style_desc)
    
    return '\n'.join(style_list)
```

#### 修改函数: `_get_style_selection_message()`
- 使用`_get_style_list_text()`生成风格列表
- 位置: 第1469-1495行

#### 更新所有风格选择提示的地方
修改了以下位置的风格选择消息,都使用`_get_style_list_text()`生成完整的风格列表:

1. **第558-576行**: 用户尝试多风格生成时的提示
2. **第676-703行**: Claude理解为多风格意图时的提示
3. **第787-805行**: 无法理解用户意图时的提示
4. **第793-811行**: 用户要求修改风格时的提示
5. **第1266-1284行**: 用户要求重新生成时的提示(COLLECT_FEEDBACK状态)
6. **第1363-1381行**: 用户要求重新生成时的提示(GENERATION_COMPLETE状态)

### 2. conversation_handler.py

- 位置: 第190-279行

## 关键改进

1. **动态生成**: 所有风格列表都从STYLE_PRESETS动态生成,确保配置和显示一致
2. **场景说明**: 明确告知用户"具体场景"风格需要场景描述,并提供3个建议场景
3. **全面覆盖**: 更新了所有可能显示风格选择的地方,确保用户在任何情况下都能看到完整的6种风格
4. **Claude理解**: 在conversation_handler的prompt中也添加了场景风格的说明,确保Claude能正确理解用户选择"具体场景"的意图
