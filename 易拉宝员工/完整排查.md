# 🔍 完整排查清单

## 当前问题
前端显示：`GET http://localhost:8000/results/banner_xxx.png net::ERR_CONNECTION_REFUSED`

说明后端返回的图片 URL 还是 `localhost:8000`。

---

## ✅ 排查步骤

### 1. 确认 Railway 环境变量

在 Railway 控制台：
- Settings → Variables
- **截图给我看所有的环境变量**

必须要有：
- `BASE_URL` = `https://你的railway域名.railway.app`
- `CLAUDE_API_KEY`
- `RUNNINGHUB_API_KEY`

### 2. 确认 Railway 已重新部署

在 Railway 控制台：
- Deployments 标签
- 查看最新部署的时间
- **必须是添加 BASE_URL 环境变量之后的时间**

如果不是，点击 **Redeploy**。

### 3. 查看 Railway 启动日志

在 Railway：
- Deployments → 最新部署 → View Logs
- 查找启动时的日志
- 看看有没有打印 BASE_URL 的值

### 4. 测试 API

在浏览器访问：
```
https://你的railway域名.railway.app/api/generation-history
```

应该返回历史记录，里面的 `image_urls` 应该是：
- ✅ `https://你的railway域名.railway.app/results/...`
- ❌ `http://localhost:8000/results/...`

**把返回的 JSON 复制给我看！**

---

## 🎯 可能的原因

### 原因 1：BASE_URL 环境变量没有真正添加

- 添加后没有点击 "Add" 按钮
- 或者添加到了错误的地方

### 原因 2：添加后没有重新部署

- Railway 不会自动重启
- 必须手动点击 Redeploy

### 原因 3：BASE_URL 值不对

- 末尾多了斜杠 `/`
- 或者协议写错了（http 而不是 https）

正确格式：
```
https://ai-agent-1-production-xxxx.up.railway.app
```

错误格式：
```
https://ai-agent-1-production-xxxx.up.railway.app/  ❌ (多了斜杠)
http://ai-agent-1-production-xxxx.up.railway.app   ❌ (http 而不是 https)
```
